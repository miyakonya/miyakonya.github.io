# ARP欺骗（ARP Spoofing）
本篇总结一下学习ARP欺骗的经验。


## 原理
根据TCP/IP协议，两台设备间要通信，需要知道对方的IP地址和MAC地址，这就要用到ARP协议。而**ARP协议本身是无认证的（stateless & trust-based），设备会无条件接受ARP响应并更新缓存，因此容易被伪造数据包欺骗。**

ARP欺骗示意图(由Gemini生成)：
![](/assets/img/arp_spoof/arp_0.jpg)

攻击者通过伪造ARP响应包，主动发送给受害者(或者直接局域网广播)，告诉受害者的设备"***某个IP(如网关IP)对应的MAC地址是xxx(攻击者指定的MAC地址)***"。
攻击者通常同时欺骗受害者和网关，使受害者认为攻击者是网关、网关认为攻击者是受害者，从而让所有进出的流量都经过攻击者。

## 危害
这是一种中间人攻击(MITM)，如果受害者用的是HTTP、FTP、Telnet等未加密的协议，攻击者可以篡改和获取受害者设备的所有网络流量。造成信息泄露。

## 实践
### 环境
- **网关**
  - IP: 192.168.8.1
  - MAC: 04-5f-a7-85-2f-10

- **攻击者**
  - IP: 192.168.8.192
  - MAC: 00:0c:29:57:c1:ef

- **受害者**
  - IP: 192.168.8.163
  - MAC: 00-0C-29-BE-3E-4B

### 一、arpspoof断网攻击
先看看受害者目前ARP路由表：

![](/assets/img/arp_spoof/arp_1.png)

这里使用`arpspoof`工具来对受害者进行arp欺骗
使用命令
```bash
arpspoof -i eth0 -t 192.168.8.163 192.168.8.1
```

- `-i`：指定网卡接口
- `-t`：指定目标主机（受害者）
- `192.168.8.1`：被伪装的IP（网关）

![](/assets/img/arp_spoof/arp_2.png)
![](/assets/img/arp_spoof/arp_3.png)
通过查看数据包的内容，可以看见攻击者伪装成网关给受害者发送ARP响应包
接着再看看受害者的ARP表：

![](/assets/img/arp_spoof/arp_4.png)

已经被篡改成了攻击者的IP
由于我的攻击设备没有开启转发，因此受害者目前是断网状态，无法对外通信
![](/assets/img/arp_spoof/arp_5.png)
我的攻击设备是Linux，执行命令
```bash
echo "1" > /proc/sys/net/ipv4/ip_forward
```
可以开启设备转发功能，此时受害者可以上网。

---

### 二、ettercap捕获数据包
`arpspoof`功能较少，一般用于ARP断网攻击，如果要截获数据，通常使用功能更强大的`ettercap`
再`ettercap`中扫描局域网，并将受害者添加为`Target 1`，网关添加为`Target 2`，然后进行ARP攻击，此时会自动开启转发，受害者能正常上网，攻击者新建终端使用`ettercap -Tq -i eth0`命令获取数据包:
![](/assets/img/arp_spoof/arp_6.png)
最下面一行得到受害者明文登录信息

## 防御
### 一、设置静态路由表
既然ARP欺骗是篡改了IP与MAC的对应关系，那么可以通过绑定这个关系来防御，Windows中可以输入命令
```bash
arp -s <IP> <MAC>
```
来设置静态arp，此时`arpspoof`的断网攻击失效，但是`ettercap`仍然可以捕获到数据包。

### 二、使用加密协议
使用HTTPS等加密协议后，虽然攻击者能够捕获，但是无法解密，也无法篡改。也可以使用VPN。

---

## 总结
ARP欺骗是一种**利用ARP协议缺乏认证的机制**而实行的中间人攻击，受到ARP欺骗后，攻击者能够截取和修改数据。
ARP欺骗除了截获数据，还能进行断网攻击，使受害者无法正常上网，这取决于攻击者设备的设置：
> 不开启转发： 断网，即拒绝服务攻击（DoS）<br>
> 开启转发： 正常上网，形成中间人攻击（MITM）
